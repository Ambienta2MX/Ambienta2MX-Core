\section{Tecnologías}
\paragraph{MogoDB} ~\\
\newline
Se hará uso de una base de datos no relacional para definir el modelo de datos y persistir la información que se obtenga de los
servicios expuestos por las instituciones.

\paragraph{Groovy} ~\\
\newline
El lenguaje de programación dinámico Groovy se empleará para el tratamiento de la información expuesta por las instituciones y
la creación de un nuevo estándar para esos metadatos.

\paragraph{Gradle} ~\\
\newline
Grade es una herramienta de construcción de proyectos que se utilizará para la administración de las dependencias y la 
automatización de tareas, como el despliegue de los módulos o la ejecución de procesos para la obtención y estandarización de los metadatos.

\paragraph{Gulp} ~\\
\newline
Gulp nos ayudará en la automatización de las tareas para la construcción y el despliegue de la aplicación en la que cualquier usuario
podrá consultar los metadatos a través de los servicios que expongamos.

\paragraph{Yeoman} ~\\
\newline
Yeoman es una herramienta que define una estructura para aplicaciones JavaScript. Será útil para generar la estructura básica de una 
aplicación que trabaje con Gulp.

\paragraph{Ember} ~\\
\newline
Es un framework para el desarrollo de single page applications. Lo utilizaremos para la estructura de la aplicación que va del 
lado del cliente ya que sus convenciones harán más escalable el desarrollo.

\paragraph{PhantomJS} ~\\
\newline
Cuando un flujo de la aplicación este completo se implementarán un conjunto de pruebas funcionales haciendo uso de PhantomJS 
para verificar si el flujo es correcto.
